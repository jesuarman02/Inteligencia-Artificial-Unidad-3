Sistema de Conflictos (Motor de Reglas)

Este módulo simula conflictos entre reglas compitiendo por activarse según sus prioridades.

Escenario:

El usuario está configurando un router nuevo. El sistema detecta varios hechos simultáneos que podrían activar reglas con distintas prioridades, como: configurar dirección IP, cambiar la contraseña del WiFi, habilitar DHCP, revisar conectividad y actualizar firmware.

Hechos Iniciales Detectados

  • El router está encendido.
  • Se detectó conexión por cable.
  • El usuario ingresó al panel de administración.

Reglas con Prioridad

Regla 1 – Prioridad Alta: Si el router tiene contraseña por defecto → pedir cambio inmediato.
Regla 2 – Prioridad Media: Si el usuario ya ingresó al panel → sugerir configurar IP LAN.
Regla 3 – Prioridad Baja: Si hay conectividad → revisar actualizaciones de firmware.
